## 📚 微信小游戏开发经验学习

### 一、小蚂蚁羊了个羊教程核心内容

**教程来源：** 微信公众号"小蚂蚁教你做游戏"
**教程系列：** 羊了个羊游戏开发实战教程（共 6 节）

#### 教程目录
1. 堆叠牌的拾取
2. 随机生成卡牌
3. 卡牌的方式和消除
4. 道具的实现
5. 制作更多的层
6. 实现快速拾取

#### 核心知识点

**1. 堆叠牌拾取（最难部分）**
- 多层堆叠布局，每层有行列网格
- 卡牌状态：可拾取（白色）/不可拾取（红色）
- 判断规律：第一层某位置能否拾取，取决于第二层四个对应位置是否有牌
- 使用数据表格记录每层牌的状态（1=有牌，0=无牌）
- 卡牌被取走后，刷新所有剩余牌的状态

**2. 随机生成卡牌**
- 10 种不同类型卡牌（动物头像）
- 数据抽象：用数字代表卡牌类型
- 生成流程：
  1. 按顺序创建卡牌类型列表
  2. 随机打乱顺序
  3. 使用打乱后的数据创建关卡
- 卡牌造型编号对应卡牌类型编号

**3. 卡牌拾取和消除**
- 拾取后放入下方插槽（卡槽）
- 卡槽满 7 张则游戏失败
- 3 张相同卡牌自动消除
- 根据当前插入牌调整其他牌位置

**4. 三种道具实现**
- **移出三张牌**：从卡槽移出前 3 张到上方临时区域
- **撤回一步**：回退上一步拾取的牌（只能回退一次）
- **随机打乱顺序**：将所有卡牌位置随机互换

**5. 快速拾取**
- 解决拾取时间间隔问题
- 优化玩家点击体验

#### 关键设计原则
- **数据抽象**：用数字/表格表示游戏状态
- **职责分离**：不同精灵负责不同功能
- **通知机制**：通过通知实现组件间通信
- **位置编号**：用数字抽象位置，方便处理

### 二、微信小游戏开发规范总结

#### 项目结构
- 核心文件：`project.config.json`、`game.json`、`game.js`
- 模块化设计，每个游戏对象有独立 `update()` 和 `render()`

#### 游戏架构
- 帧循环：`requestAnimationFrame` → `loop()` → `update()` + `render()`
- 数据管理：DataBus 类统一管理状态和分数
- 对象池：统一管理游戏对象生命周期

#### 排行榜实现
- 主域：`wx.setUserCloudStorage()` 上传分数
- 开放数据域：`wx.getFriendCloudStorage()` 获取好友数据
- 主域和子域通过 `postMessage` 通信（单向）
- 配置在 MP 管理后台完成

#### 踩坑总结
| 坑 | 解决方案 |
|---|---|
| 开放数据域发布缺失文件 | 从官方 Demo 复制 |
| 子域无法向主域发消息 | 关闭按钮做在主域 |
| JS 文件未压缩 | 发布 HTML5 版本触发压缩 |
| 广告审核被打回 | 广告位居中，激励广告有提示 |

## 📚 消除游戏系列教程深入学习（2026-05-04 下午）

### 一、三消游戏查找算法

**核心原理：**
- 从点击位置开始，查找水平/竖直方向的连续3个相同图标
- 使用递归查找算法
- 匹配列表管理：记录所有匹配的图标位置

**实现步骤：**
1. 获取点击位置的行列号
2. 查找水平方向：左、右两个方向
3. 查找竖直方向：上、下两个方向
4. 将匹配的图标加入匹配列表
5. 执行消除操作

**关键代码逻辑：**
```
递归查找函数：
- 如果当前图标 == 目标图标 && 未被查找过
  - 加入匹配列表
  - 继续查找相邻位置（左、上、右、下）
```

### 二、点消游戏查找算法

**核心原理：**
- 点击任意图标，周围所有相同图标同时消除
- 递归查找：找到一个相同图标，以它为中心继续查找
- 只需查找一轮（相邻2个即可消除）

**查找过程：**
1. 从点击位置开始
2. 依次查找左、上、右、下四个方向
3. 发现相同图标 → 递归查找
4. 直到所有相邻图标查找完毕

**比喻理解：**
- 查找过程像找树的每一个分支
- 从主干开始，遇到树叉继续向前找
- 最终回到主干时，所有分叉都找遍

### 三、连线消除游戏

**核心原理：**
- 玩家手动划线匹配
- 匹配判断 + 连线创建
- 连接线创建：根据两个点坐标设置

**实现步骤：**
1. 获取两个图标的索引
2. 计算索引对应的行列号
3. 根据行列号计算屏幕坐标 (x,y)
4. 创建连接线
5. 执行消除操作

**关键函数：**
- `获取两个点位置`：利用索引计算屏幕坐标
- `设置连接线`：根据两点坐标创建连线

### 四、图标下落原理

**核心原理：**
- 消除后上方图标下落填补空位
- 连锁消除机制
- 下落动画 + 新图标生成

**实现逻辑：**
1. 消除图标后，上方图标下移
2. 检查是否有新的匹配
3. 如有匹配 → 继续消除（连锁）
4. 无匹配 → 生成新图标
