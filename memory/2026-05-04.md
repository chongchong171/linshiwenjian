#### 核心知识点

**1. 堆叠牌拾取（最难部分）**
- 多层堆叠布局，每层有行列网格
- 卡牌状态：可拾取（白色）/不可拾取（红色）
- 判断规律：第一层某位置能否拾取，取决于第二层四个对应位置是否有牌
- 使用数据表格记录每层牌的状态（1=有牌，0=无牌）
- 卡牌被取走后，刷新所有剩余牌的状态

**2. 随机生成卡牌**
- 10 种不同类型卡牌（动物头像）
- 数据抽象：用数字代表卡牌类型
- 生成流程：
  1. 按顺序创建卡牌类型列表
  2. 随机打乱顺序
  3. 使用打乱后的数据创建关卡
- 卡牌造型编号对应卡牌类型编号

**3. 卡牌拾取和消除**
- 拾取后放入下方插槽（卡槽）
- 卡槽满 7 张则游戏失败
- 3 张相同卡牌自动消除
- 根据当前插入牌调整其他牌位置

**4. 三种道具实现**
- **移出三张牌**：从卡槽移出前 3 张到上方临时区域
- **撤回一步**：回退上一步拾取的牌（只能回退一次）
- **随机打乱顺序**：将所有卡牌位置随机互换

**5. 快速拾取**
- 解决拾取时间间隔问题
- 优化玩家点击体验

#### 关键设计原则
- **数据抽象**：用数字/表格表示游戏状态
- **职责分离**：不同精灵负责不同功能
- **通知机制**：通过通知实现组件间通信
- **位置编号**：用数字抽象位置，方便处理

### 一、三消游戏查找算法

**核心原理：**
- 从点击位置开始，查找水平/竖直方向的连续3个相同图标
- 使用递归查找算法
- 匹配列表管理：记录所有匹配的图标位置

**实现步骤：**
1. 获取点击位置的行列号
2. 查找水平方向：左、右两个方向
3. 查找竖直方向：上、下两个方向
4. 将匹配的图标加入匹配列表
5. 执行消除操作

**关键代码逻辑：**
```
递归查找函数：
- 如果当前图标 == 目标图标 && 未被查找过
  - 加入匹配列表
  - 继续查找相邻位置（左、上、右、下）
```

### 二、点消游戏查找算法

**核心原理：**
- 点击任意图标，周围所有相同图标同时消除
- 递归查找：找到一个相同图标，以它为中心继续查找
- 只需查找一轮（相邻2个即可消除）

**查找过程：**
1. 从点击位置开始
2. 依次查找左、上、右、下四个方向
3. 发现相同图标 → 递归查找
4. 直到所有相邻图标查找完毕

**比喻理解：**
- 查找过程像找树的每一个分支
- 从主干开始，遇到树叉继续向前找
- 最终回到主干时，所有分叉都找遍

### 三、连线消除游戏

**核心原理：**
- 玩家手动划线匹配
- 匹配判断 + 连线创建
- 连接线创建：根据两个点坐标设置

**实现步骤：**
1. 获取两个图标的索引
2. 计算索引对应的行列号
3. 根据行列号计算屏幕坐标 (x,y)
4. 创建连接线
5. 执行消除操作

**关键函数：**
- `获取两个点位置`：利用索引计算屏幕坐标
- `设置连接线`：根据两点坐标创建连线

### 四、图标下落原理

**核心原理：**
- 消除后上方图标下落填补空位
- 连锁消除机制
- 下落动画 + 新图标生成

**实现逻辑：**
1. 消除图标后，上方图标下移
2. 检查是否有新的匹配
3. 如有匹配 → 继续消除（连锁）
4. 无匹配 → 生成新图标

#### 核心问题一：创建层叠布局

**2.1 基本思路**
- 一层一层地创建，先创建第一层，再创建第二层
- 示例：第一层 3 行 5 列，第二层 3 行 4 列
- 第二层压在第一层上方

**2.2 计算网格起点位置**
- 已知：每个格子宽高、行数、列数、网格中心坐标
- 推算：整个表格左上角位置作为创建牌的起点

**2.3 计算每张牌的位置**
- 起点坐标 + 当前牌行列号 + 每个格子宽高
- 代入位置计算公式得出精确坐标

**2.4 第二层牌的偏移**
- 第二层牌整体向右和向上各偏移半个格子距离
- 第二层起点位置 = 第一层起点位置 + 半格偏移量

**2.5 实现逻辑**
1. 场景启动：计算第一层起点 (x, y)，再计算第二层起点 (x, y)
2. 创建第一层卡牌：全局变量表示当前层级（1），双重循环（行 + 列）
3. 等待 0.5 秒（让玩家看到创建顺序）
4. 创建第二层卡牌：层级变量设为 2，双重循环

**2.6 卡牌创建后的位置设置**
- 层级为 1：使用第一层起点位置计算坐标
- 层级为 2：使用第二层起点位置计算坐标
- 行列号代入计算公式，设置最终位置

#### 核心问题二：判断牌能否被拾取（⭐ 核心难点）

**3.1 问题描述**
- 下层牌可能被上层牌压住
- 被压住的牌不能点击拾取
- 只有上方没有牌压住的牌才能拾取
- 不同位置的牌上方压的牌数量不同（1 张、2 张或 4 张）

**3.2 核心规律（⭐⭐⭐⭐⭐）**
已知第一层某张牌的行列号 (r, c)，需要检查第二层表格中以下四个位置是否有牌：
- (r, c)
- (r+1, c)
- (r, c-1)
- (r+1, c-1)

判断规则：
- 四个位置都没有牌 → 该牌可以被拾取
- 任何一个位置有牌 → 该牌不能被拾取（被压住）
- 超出第二层表格边界 → 视为没有牌压住

**3.3 举例说明**
- 例子 1：第一层牌位于 (3, 1) → 检查 (3,1)、(4,1)、(3,0)、(4,0)
- 例子 2：第一层牌位于 (2, 3) → 检查 (2,3)、(3,3)、(2,2)、(3,2)
- 例子 3：第一层牌位于 (2, 5) → 检查 (2,5)、(3,5)、(2,4)、(3,4)

**3.5 实现逻辑**

数据结构：
- 一层表格：3 行 5 列，每个数据初始为 1（表示有牌）
- 二层表格：3 行 4 列，每个数据初始为 1（表示有牌）
- 牌被拿走后，对应位置数据设为 0

卡牌被按下的逻辑：
1. 判断该牌是否可拾取
2. 不能拾取 → 牌抖动（视觉反馈）
3. 可以拾取 → 对应表格位置数据设为 0
4. 发送通知，所有剩余卡牌刷新状态

卡牌刷新状态的逻辑：
1. 判断检查位置是否在表格边界内
2. 位置在边界内，查看数据是 0 还是 1
   - 数据为 1 → 该位置有牌 → 不可拾取，牌颜色变红色，**跳出检查逻辑**
3. 四个位置都检查完毕且都没有牌 → 可拾取，牌颜色变白色

**关键优化：** 检查四个位置时，一旦发现任何一个位置有牌压住，就**立即停止检查**（跳出逻辑），因为只要有一个位置被压住就够了，无需继续判断其余位置。

#### 四、核心算法深度解析（⭐⭐⭐⭐⭐）

**4.1 堆叠偏移规律**
- 第二层牌整体向右和向上各偏移半个格子距离
- 第二层起点位置 = 第一层起点位置 + 半格偏移量
- 这意味着：第二层的每个牌覆盖第一层的 2x2 区域

**4.2 遮挡判断核心算法（必须掌握）**
已知第一层某张牌的行列号 (r, c)，需要检查第二层表格中以下四个位置是否有牌：
- (r, c)
- (r+1, c)
- (r, c-1)
- (r+1, c-1)

判断规则：
- 四个位置都没有牌 → 该牌可以被拾取
- 任何一个位置有牌 → 该牌不能被拾取（被压住）
- 超出第二层表格边界 → 视为没有牌压住

**4.3 关键优化：提前跳出**
检查四个位置时，一旦发现任何一个位置有牌压住，就**立即停止检查**（跳出逻辑），因为只要有一个位置被压住就够了，无需继续判断其余位置。

**4.4 数据结构设计**
- 使用两个表格（数组）存储每层牌的状态
- 一层表格：3 行 5 列，每个数据初始为 1，表示该位置有牌
- 二层表格：3 行 4 列，每个数据初始为 1，表示该位置有牌
- 牌被拿走后，将对应位置的数据设为 0

**4.5 卡牌被按下的逻辑流程**
1. 首先判断该牌是否可拾取
2. 如果不能拾取 → 让牌抖动一下（视觉反馈）
3. 如果可以拾取 → 将该牌对应表格位置的数据设为 0
4. 发送通知，让所有剩余卡牌刷新状态

**4.6 卡牌刷新状态的逻辑**
1. 先判断要检查的位置是否在表格边界内（行号 > 0 且 < 表格行数，列号 > 0 且 < 表格列数）
2. 如果位置在边界内，查看该位置的数据是 0 还是 1
   - 数据为 1 → 该位置有牌 → 当前牌不可拾取，将"可拾取"变量设为 0，牌颜色变为红色，直接跳出整个检查逻辑
3. 四个位置都检查完毕且都没有牌 → 当前牌可拾取，将"可拾取"变量设为 1，牌颜色变为白色

#### 七、系列 2：随机生成卡牌深度学习（2026-05-04 22:24）

**教程来源：** 小蚂蚁教你做游戏 - 羊了个羊游戏制作教程·系列 2：随机生成卡牌

##### 7.1 核心理论：数据抽象
- 将 10 个小动物头像编号为 1~10
- 用数字代表不同的头像
- 所有数据运算和规律查找都只需操作数字，无需关注图标本身
- 最后通过数字还原为对应的图片（数据抽象的还原过程）

##### 7.2 保证关卡可通关的两个必要条件（⭐⭐⭐⭐⭐）
1. **卡牌总数必须是 3 的倍数**
   - 三消游戏是每三张相同牌消除一组
   - 如果总数不是 3 的倍数（如 14 张），最后一定会剩余无法消除的牌
2. **每一个类别的卡牌数量也必须是 3 的倍数**
   - 例如如果有 9 个猴子头像，可以分三组消除
   - 但如果只有 8 个，最后一定会有 2 个无法消除

##### 7.3 随机生成卡牌的完整流程
以 10 种动物头像（编号 1~10）、关卡中出现 5 种类型、共 27 张卡牌为例：

**步骤 1：随机抽取类型编号**
- 从 1~10 这 10 个数字中，随机抽取 5 个不重复的数字
- 例如抽到：2、4、5、7、9

**步骤 2：按类型依次生成，每类 3 张**
- 遍历这 5 个数字，每个数字连续生成 3 个：
  - 生成三个 2：2, 2, 2
  - 生成三个 4：4, 4, 4
  - 生成三个 5：5, 5, 5
  - 生成三个 7：7, 7, 7
  - 生成三个 9：9, 9, 9
- 第一轮共生成 15 个数字
- 当遍历到列表末尾时，从头重新开始，继续按同样规则生成
- 直到凑满 27 个数字
- **关键点：** 因为每次都是 3 个 3 个生成的，所以每个类型的数量一定是 3 的倍数，满足通关条件 2

**步骤 3：随机打乱序列**
- 上述序列是整齐排列的（相同数字总是相邻），游戏会太简单
- 需要将整个序列随机打乱顺序
- 打乱后既保证了可通关性，又增加了随机性

##### 7.4 具体实现要点
- **统计卡牌总数：** 自动遍历第一层表格和第二层表格，统计值为 1 的格子数量
- **验证卡牌总数是否为 3 的倍数：** 使用取余运算判断：如果 卡牌总数 % 3 == 0，则满足条件
- **从 1~10 中随机抽取 5 个不重复数字：** 利用列表操作，从 1~10 的十个数字中抽取指定数量的不重复数字

  - 使用一个迭代变量（从 1 开始递增）
  - 每次向列表插入当前索引对应的数字
  - 当迭代变量 % 3 == 0 时，说明已经插入 3 个相同数字，将索引 +1（切换到下一个类型）
  - 当索引超过"随机牌索引"列表的长度时，将索引重置为 1（从头循环）
- **随机打乱列表：** 使用"随机打乱列表"功能
- **创建卡牌时的修改：**
  - 创建卡牌前，将"卡牌编号"设置为 1
  - 每创建一张卡牌，卡牌编号 +1
  - 设置类型：根据卡牌自己的编号，从"随机牌列表"中获取对应位置的数字，作为该卡牌的类型
  - 切换造型：将卡牌切换到与类型编号对应的造型（数字 1~10 对应造型 1~10）

#### 八、系列 3：卡牌拾取和消除深度学习（2026-05-04 22:27）

**教程来源：** 小蚂蚁教你做游戏 - 羊了个羊游戏制作教程·系列 3：卡牌拾取和消除

##### 8.1 核心数据模型 — 卡牌类型编号化
- 游戏中共有 10 种小动物头像，将它们编号为 1～10 的整数
- 之后所有逻辑都使用数字来代表卡牌类型，而非直接操作图标
- 例如：卡牌槽中的牌可以表示为 [5, 5, 3, 1]

##### 8.2 卡牌插入卡牌槽的逻辑（⭐⭐⭐⭐⭐）
当玩家点击上方堆叠区的一张牌时，该牌会飞入下方的卡牌槽。插入时有三种情况：

**情况一：卡槽中没有与插入牌相同的牌**
- 新牌直接追加到卡牌槽末尾
- 示例：卡槽为 [1, 2, 3]，插入类型 5 → 结果 [1, 2, 3, 5]

**情况二：卡槽中已有一个与插入牌相同类型的牌**
- 新牌插入到那个相同牌的后面，其后的牌依次后移一个位置
- 示例：卡槽为 [5, 3, 1]，插入类型 5 → 结果 [5, 5, 3, 1]

**情况三：卡槽中已有两个与插入牌相同类型的牌**
- 新牌插入到第二个相同牌的后面，凑成三个相同牌触发消除，其后的牌后移
- 示例：卡槽为 [5, 5, 3, 1]，插入类型 5 → 结果 [5, 5, 5] 凑齐三个消除，剩余 [3, 1]

##### 8.3 卡牌插入算法 — 标记位置法（⭐⭐⭐⭐⭐）
**核心思想：** 通过一轮遍历检查，找到一个"标记位置"，然后新牌插入到标记位置 + 1 处。

**算法步骤：**
1. 设置局部变量 标记位置 = 0（默认值）
2. 遍历卡牌槽中的每一张牌：
   - 如果当前牌的类型 == 待插入牌的类型 → 将 标记位置 更新为当前位置
   - 如果当前牌的类型 != 待插入牌的类型 → 标记位置不变，继续检查
3. 遍历结束后判断：
   - 如果 标记位置 > 0：说明卡槽中存在相同类型的牌，新牌插入位置 = 标记位置 + 1
   - 如果 标记位置 == 0：说明卡槽中没有相同类型的牌，新牌插入位置 = 列表项目数 + 1（即末尾）
4. 插入新牌后，将卡牌类型数字也插入到对应的"插槽牌类型列表"中
5. 设置全局变量 当前卡牌的插入位置 为新牌的插入位置（供其他卡牌判断是否需要调整位置）
6. 向所有插槽卡牌发送通知 "插入卡牌的位置调整"

**插入后的位置调整通知：**
- 当新牌插入后，卡槽中所有位置 >= 当前卡牌插入位置的已有牌，需要将自己的位置向右移动一位（位置编号 +1）
- 然后通过动画移动到目标位置

##### 8.4 卡牌按下的实现修改
**原来的逻辑：** 卡牌被按下时直接删除

**修改后的逻辑：**
- 不再直接删除卡牌
- 而是克隆一个新的"插槽卡牌"（用于在卡牌槽中显示和处理逻辑）
- 原卡牌从堆叠区移除

##### 8.5 插槽卡牌被创建时的流程
1. 设置自身的类型变量和图标
2. 执行遍历检查过程（标记位置法），确定插入位置
3. 将自己插入到卡牌槽的合适位置
4. 将类型数字插入到"插槽牌类型列表"对应位置
5. 记录全局变量 当前卡牌的插入位置
6. 发送通知"插入卡牌的位置调整"（让其他牌调整位置）
7. 将自己移动到目标位置（动画）
8. 发送通知"卡牌槽进行消除计算"

##### 8.6 消除逻辑 — 三消检查（⭐⭐⭐⭐⭐）
当一张新牌插入卡牌槽后，需要检查卡槽中是否有三个相同类型的牌可以消除。

**关键特性：** 在"羊了个羊"中，卡牌槽同一时刻最多只能消除一组三张相同的牌，不可能同时存在两组三消。因此一旦找到一组满足三消的条件，检查即可终止。

**消除检查算法：**
使用两个变量：
- **检查类型**（当前正在检查的卡牌类型）
- **匹配数量**（当前类型已匹配的卡牌数量）

**步骤：**
1. 前置判断：如果 插槽牌类型列表的项目数 <= 2，直接返回（不足三张牌不可能消除）
2. 循环遍历 项目数 - 2 次（因为从第 i 个位置开始，需要往后看两个位置，所以只需要遍历到倒数第三个元素）
3. 对于每个起始位置 i：
   - 记录 检查类型 = 列表第 i 项
   - 匹配数量 = 1
   - 用内层循环检查位置 i+1 和 i+2：
     - 如果类型相同 → 匹配数量 += 1
     - 如果类型不同 → 直接跳出（无需继续检查，因为相同类型的牌必须连续排列）
4. 检查完毕后，如果 匹配数量 >= 3 → 满足三消条件，执行消除处理

**消除处理步骤：**
1. 将当前满足消除的类型数字记录到全局变量 需要消除的卡牌类型（供各卡牌判断自己是否该被消除）
2. 从"插槽牌类型列表"中删除对应的三项
3. 向所有插槽卡牌发送通知 "进行消除处理"
4. 设置全局变量 当前卡牌的消除位置 = 遍历时的迭代变量 i 的值（即三张相同牌中第一张的位置，也即标记位置）
5. 向所有插槽卡牌发送通知 "消除卡牌的位置调整"

##### 8.7 两个消除相关通知的处理逻辑
这两个通知都在插槽卡牌上处理。

**通知 1：消除处理**
- 逻辑简单：判断当前卡牌的类型是否 == 需要消除的卡牌类型
- 如果相同 → 删除该卡牌（执行消除）

**通知 2：消除卡牌的位置调整**
- 消除后，卡槽中剩余的牌需要前移填补空位
- 判断当前卡牌的位置是否 >= 当前卡牌的消除位置
- 如果是 → 将该卡牌的位置向前移动 3 个位置（位置编号 -3）
- 通过动画将卡牌移动到目标位置

##### 8.8 完整流程总结
1. 玩家点击堆叠区卡牌
2. 原卡牌移除，克隆一个新的插槽卡牌
3. 插槽卡牌通过标记位置法确定插入位置
4. 新牌插入卡牌槽，类型数字同步插入列表
5. 通知所有插槽卡牌进行插入位置调整（插入点之后的牌右移一位）
6. 新牌动画移动到目标位置
7. 通知卡牌槽进行消除计算
8. 遍历检查是否存在三消
9. 若满足三消：
   - 记录消除类型
   - 从列表删除三项
   - 通知所有插槽卡牌执行消除
   - 记录消除位置
   - 通知所有插槽卡牌执行消除位置调整（消除点之后的牌前移三位）
10. 剩余卡牌动画移动到新位置

| 变量名 | 类型 | 作用 |
|--------|------|------|
| 标记位置 | 局部变量 | 遍历检查时记录最后一个与插入牌类型相同的位置 |
| 插槽牌类型列表 | 全局列表 | 存储卡牌槽中所有牌的类型数字，与实际卡牌一一对应 |

#### 九、系列 4：三种道具的实现深度学习（2026-05-04 22:36）

**教程来源：** 小蚂蚁教你做游戏 - 羊了个羊游戏制作教程·系列 4：三种道具的实现

##### 9.1 核心设计思想 — 职责分离（⭐⭐⭐⭐⭐）
游戏中看似只是一张牌在移动，实际上使用了三种不同类型的卡牌对象来分别处理不同职责：
- **卡牌：** 上方层叠区域中显示的牌，负责展示和被拾取
- **插槽卡牌：** 飞入下方卡槽后的牌，负责在卡槽中的排列和消除判断
- **移出卡牌：** 从卡槽移到上方的牌，负责暂存和回填
- 每种卡牌只关心自己的逻辑，互不干扰，便于拆分和组合

##### 9.2 道具一：移出三张牌
**功能：** 将下方卡槽中的前几张牌移到卡槽上方暂存；后续可以点击移出的牌将其放回卡槽。

**两种情况：**
1. **卡槽中牌数 ≤ 3 张：** 直接将所有牌移到上方，移完后卡槽为空。实际移出数量 = 当前卡槽中的牌数。
2. **卡槽中牌数 > 3 张：** 移出前三张牌到上方，剩余的牌需要向左移动三个位置进行位置调整。

**实现逻辑（按钮上的积木）：**
1. 设置移出卡牌数量为 3
2. 判断当前卡槽中牌数是否小于 3，如果是则将移出数量改为实际牌数
3. 发送通知给所有插槽卡牌，让它们执行"移出卡牌"操作
4. 设置移出卡牌的位置编号为 1
5. 使用循环，克隆指定数量的移出卡牌
6. 使用循环，从卡槽类型列表中删除前几项（因为牌已移出）
7. 发送通知，让剩余插槽卡牌进行位置调整

**移出卡牌自身的逻辑：**
- **被创建时：** 设置类型和图标，将自己移动到上方指定位置
- **被按下时：** 创建一张新的插槽卡牌放回卡槽，然后删除自身

**插槽卡牌收到"移出卡牌"通知时：**
- 判断自己的位置是否 ≤ 移出数量，如果是则删除自身（因为这张牌要被移走）

**插槽卡牌收到"位置调整"通知时：**
- 位置值减去移出数量，然后用动画移动到新位置

##### 9.3 道具二：撤回一步
**功能：** 将最后一步拾取的牌从卡槽中退回到它原来在层叠区的位置。只能撤回一步，不能连续撤回多步。

**核心难点：** 撤回时必须知道这张牌原来的所有位置信息，包括行号、列号、类型、层级，才能准确放回原位。因此需要一组变量来记录撤回信息。

**所需变量：**
- 撤回卡牌的行号、列号、类型、插槽位置、层级等

**实现逻辑（按钮上的积木）：**
1. 点击按钮后发送两个通知：
   - 通知所有插槽卡牌"撤回卡牌"
   - 通知所有卡牌"刷新卡牌"（重新计算可拾取状态）

**插槽卡牌收到"撤回卡牌"通知时：**
- 判断自己的位置是否等于"撤回卡牌插槽位置"（这个位置变量在每次创建新插槽卡牌时记录，始终指向最后加入的那张牌）
- 如果匹配：
  - 从插槽类型列表中删除对应项
  - 设置"撤回卡牌"标记为 1
  - 克隆一张新的卡牌（放回上方）
  - 发送通知让剩余插槽卡牌调整位置
  - 最后删除自身

**卡牌克隆时的逻辑修改：**
- 增加判断：是否为撤回的卡牌
  - 如果不是撤回卡牌：走原有逻辑
  - 如果是撤回卡牌：使用之前记录的撤回变量（行号、列号、层级等）来设置新卡牌的位置信息，将其放回原来的位置

**插槽卡牌收到"位置调整"通知时：**
- 如果自己的位置 > 撤回卡牌的位置，则位置减 1，向前移动一格

**为什么需要"刷新卡牌"通知：**
- 因为游戏是层级堆叠的，上层牌被拾取后下层牌变为可拾取
- 撤回操作会把牌放回去，原本被"解放"的下层牌需要重新变为不可拾取状态
- 所以必须刷新所有卡牌的状态

##### 9.4 道具三：随机打乱
**功能：** 将当前上方所有剩余牌的位置进行随机打乱，在不改变牌的层级关系的前提下重新排列，给困境中的游戏带来转机。

**理论基础：**
- 每张牌的位置信息由行号、列号、层级三部分组成
- 为方便理解，将每个位置抽象为一个数字编号
- 打乱的过程如下：
  1. 将所有牌的位置编号记录到一个列表中（如 [1,2,3,4,5,6,7,8,9]）
  2. 将这个列表随机打乱顺序（如得到 [7,6,9,4,1,3,5,8,2]）
  3. 从头开始，将乱序列表中的位置编号重新设置到每张牌上
  4. 所有牌根据新的位置编号移动到对应位置

**所需列表：**
- 卡牌行号列表、卡牌列号列表、卡牌层级列表、随机乱序列表

**实现逻辑（按钮上的积木）：**
1. 清空所有列表中的旧数据
2. 发送通知给所有卡牌"记录卡牌的位置"——每张牌将自己的行号、列号、层级分别插入到对应列表中
3. 使用循环创建一个整齐的乱序列表（如剩余 10 张牌则创建 [1,2,3,...,10]）
4. 将乱序列表随机打乱
5. 将卡牌编号设置为 1，发送通知"随机设置卡牌的位置"——每张牌按编号从乱序列表中取出对应位置信息，设置为自己的新行号、列号、层级
6. 设置随机打乱层级为 1，使用循环逐层发送通知"随机打乱卡牌"——先移动第一层牌，再移动第二层牌
7. 发送通知"刷新卡牌"——重新计算所有牌的可拾取状态

**为什么要逐层调整位置：**
- 在 2D 游戏中，层级高的图片会覆盖层级低的图片
- 随机打乱后，原本第一层的牌可能跑到第二层的位置，反之亦然
- 为保证第二层牌永远显示在第一层牌上方，必须按层级从低到高依次调整：
  - 先调整第一层牌的位置（此时第一层牌位于图片最上层）
  - 再调整第二层牌的位置（第二层牌又会置于最上层）
- 这样就能保证最终的显示层级正确

**卡牌收到"随机打乱卡牌"通知时：**
- 判断自己的层级是否等于当前处理层级，如果是则将自己移动到新位置，并将自身图片层级设为最上层

##### 9.5 三种道具对比

| 道具 | 功能 | 关键机制 | 核心通知 |
|------|------|---------|---------|
| 移出三张牌 | 将卡槽前 3 张牌暂存到上方 | 区分两种情况（≤3 和 >3），移出后剩余牌位置调整 | 移出卡牌 + 位置调整 |
| 撤回一步 | 将最后拾取的牌退回原位 | 记录撤回信息变量，克隆新卡牌放回原位，刷新所有牌状态 | 撤回卡牌 + 刷新卡牌 |
| 随机打乱 | 随机打乱所有剩余牌位置 | 位置列表记录→打乱→重设位置，逐层移动保证层级正确 | 记录位置 + 设置位置 + 打乱卡牌 + 刷新卡牌 |

**1. 每层边界收缩逻辑（第21课）⭐⭐⭐⭐⭐**
```
原理：
- 每层比上一层缩小边界
- 形成自然的堆叠效果
- 需要计算每层的块数量和位置

实现要点：
- 计算每层的边界收缩量
- 根据收缩量调整每层的块数量
- 确保上层块正确遮挡下层块
```

**2. 坐标计算（第22-23课）⭐⭐⭐⭐⭐**
```
随机区域：
- 不规则分布
- 需要随机生成坐标
- 确保不重叠

有规律区域：
- 网格分布
- 精确计算每个块的位置
- 保持对齐
```

**3. 堆叠关系处理（第24课）⭐⭐⭐⭐⭐**
```
原理：
- 块之间的层级关系管理
- 上层块遮挡下层块
- z-index 动态管理

实现要点：
- 根据层级计算 z-index
- 确保上层块显示在下层之上
- 处理遮挡关系
```

**4. 槽位管理（第28课）⭐⭐⭐⭐⭐**
```
原理：
- 底部7格槽位
- 点击植物放入槽位
- 凑3消

实现要点：
- 槽位数量限制（7格）
- 槽位内排序
- 满槽判断
- 消除逻辑
```

**5. 消除逻辑（第29课）⭐⭐⭐⭐⭐**
```
原理：
- 槽位内3个相同植物消除
- 消除后槽位重新排序
- 检查连锁消除

实现要点：
- 匹配查找算法
- 消除动画
- 槽位更新
- 连锁检测
```
