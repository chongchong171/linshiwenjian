# 绿了个植 - 云开发排行榜方案

## 一、微信云开发基础

### 1. 云开发环境
- 在微信开发者工具中开通云开发服务
- 创建云环境（最低收费 19.9 元/月/环境）
- 小游戏端初始化：`wx.cloud.init({ env: 'your-env-id' })`

### 2. 云函数示例
```javascript
// 云函数入口文件
const cloud = require('wx-server-sdk')
cloud.init({ env: cloud.DYNAMIC_CURRENT_ENV })

exports.main = async (event, context) => {
  const wxContext = cloud.getWXContext()
  return {
    openid: wxContext.OPENID,
  }
}
```

### 3. 前端调用云函数
```javascript
wx.cloud.callFunction({
  name: 'getOpenid',
  success: res => {
    console.log('用户openid：', res.result.openid)
  }
})
```

## 二、排行榜实现方案

### 1. 官方排行榜（推荐优先使用）

**核心 API：**
- `wx.setUserCloudStorage()` - 上传当前用户分数
- `wx.getFriendCloudStorage()` - 获取好友分数（开放数据域）

**配置步骤：**
1. 完成规范格式的后台数据上报
2. 登录 MP 管理后台 → 游戏能力地图-社交能力-微信排行榜配置
3. 配置字段：
   - 排行榜唯一标识（key 值）
   - 排行榜名称
   - 更新周期
   - 排行榜数据类型（整数型/枚举值）
   - 排行榜顺序（升序/降序）
   - 数据单位后缀
4. 提交审核通过后生效

**value 格式：**
```javascript
{
  score: Int32,        // 分数
  update_time: Int64   // 更新时间戳
}
```

---

### 2. 自定义好友排行榜

**实现步骤：**
1. 主域：`wx.setUserCloudStorage()` 上传分数
2. 开放数据域：`wx.getFriendCloudStorage()` 获取好友数据
3. 主域和子域通过 `postMessage` 通信（单向：主→子）
4. 子域自行绘制排行榜界面

**注意事项：**
- 关闭按钮必须做在主域
- 子域无法主动发消息给主域
- 发布开放数据域项目会缺失文件，必须从官方 Demo 复制

## 三、数据存储方案

### 1. 云数据库集合设计

| 集合名 | 用途 | 字段 |
|--------|------|------|
| `users` | 用户档案 | openid, nickname, avatar, create_time |
| `scores` | 分数记录 | openid, score, level, create_time |
| `gardens` | 花园数据 | openid, plants, level, decorations |
| `levels` | 关卡数据 | level_id, theme, difficulty, config |

### 2. 云函数设计

| 函数名 | 功能 | 参数 | 返回 |
|--------|------|------|------|
| `getUserInfo` | 获取用户信息 | openid | 用户档案 |
| `updateScore` | 更新分数 | openid, score, level | 成功/失败 |
| `getRankList` | 获取排行榜 | type, limit | 排行榜列表 |
| `getGardenData` | 获取花园数据 | openid | 花园信息 |
| `updateGarden` | 更新花园 | openid, data | 成功/失败 |

## 四、开放数据域实现

### 1. 项目结构
```
minigame/
├── game.js              # 主域入口
├── game.json            # 游戏配置
├── openDataContext.js   # 开放数据域入口
├── open-data/           # 开放数据域子项目
│   ├── index.js
│   ├── leaderboard.js
│   └── render.js
└── src/
    ├── scenes/
    ├── components/
    └── utils/
```

### 2. 主域与子域通信

**主域发送消息：**
```javascript
const openDataContext = wx.getOpenDataContext()
openDataContext.postMessage({
  type: 'updateRank',
  data: { score: 100 }
})
```

**子域接收消息：**
```javascript
wx.onMessage((data) => {
  if (data.type === 'updateRank') {
    updateRankList(data.data.score)
  }
})
```

### 3. 排行榜绘制

**使用轻量级渲染引擎：**
- 推荐使用 Layout 等轻量级渲染引擎
- 开放数据域的逻辑是独立的
- 可以复用主项目的图片资源

## 五、踩坑总结

| 坑 | 解决方案 |
|---|---|
| 发布开放数据域缺失文件 | 从官方 Demo 复制缺失文件 |
| 子域无法向主域发消息 | 关闭按钮做在主域，主域通知子域 |
| JS 文件未压缩 | 发布 HTML5 版本触发压缩 |
| 云函数无状态 | 不要在云函数中做缓存，每次调用都是新实例 |
| 共享环境 openid | 使用 `wxContext.FROM_OPENID` 而不是 `wxContext.OPENID` |
