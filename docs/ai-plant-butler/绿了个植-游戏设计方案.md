> 基于简游科技（羊了个羊/砖了个砖/开局托儿所/拔了个罐/干了个饭）、抓大鹅、月之村等爆款小游戏的核心机制提炼，结合AI植物管家的植物科普定位，打造一款有差异化竞争力的社交休闲消除小游戏。

## 二、核心玩法设计

### 2.1 基础消除机制

**玩法类型：滑动相邻消除（参考砖了个砖）**

**操作流程：**
1. 进入关卡 → 看到一盘杂乱排列的植物图案（多层叠放）
2. **滑动屏幕**拖动一个植物到相邻空位
3. 当两个及以上相同品种的植物在**同一直线**上相连 → 点击它们
4. 植物被消除，上方植物自动下落填补空位
5. 全部消除 → 通关

**与羊了个羊的区别：**

| 维度 | 羊了个羊 | 绿了个植 |
|------|---------|---------|
| 消除方式 | 点击选砖放入底部7格槽位，凑3消 | 滑动到相邻位置，直线2+消 |
| 槽位限制 | 底部7格，满了就失败 | 无槽位限制，只有消除机会限制 |
| 视觉呈现 | 平面卡片叠放 | 花盆/泥土场景，植物从土里长出来 |
| 互动方式 | 纯点击 | 滑动+摇手机+浇水 |

**为什么选滑动相邻消除：**
- 差异化好，折叠消除已经被羊了个羊占位
- 策略性强，玩家需要规划消除顺序
- 与植物"移栽""嫁接"的自然动作契合
- 2+消比3消更容易上手，降低入门门槛

### 2.3 断崖式难度设计（核心上瘾机制）

**来源：简游科技所有爆款的核武器——第一关闭眼过，第二关地狱级**

#### 第一关设计（入门关）

| 参数 | 值 |
|------|-----|
| 植物种类 | 4种 |
| 每种数量 | 3个 |
| 总元素数 | 12个 |
| 布局 | 单层平铺，无遮挡 |
| 通关时间 | 3-5秒 |
| 通关率 | 100% |

**布局示例（多肉花园第一关）：**
```
[仙人掌] [芦荟] [景天] [生石花]
[仙人掌] [芦荟] [景天] [生石花]
[仙人掌] [芦荟] [景天] [生石花]
```
玩家随便滑动就能连成直线，3秒消除完。

**通关反馈：**
- 植物"嗖"地缩回土里，花盆冒出一颗小芽
- 文字："入门了！🌱"
- 音效：轻快的"叮"声

---

#### 第二关设计（地狱关）

| 参数 | 值 |
|------|-----|
| 植物种类 | 8种 |
| 每种数量 | 3个 |
| 总元素数 | 24个 |
| 布局 | 3层叠放，部分遮挡 |
| 通关时间 | 3-8分钟 |
| 目标通关率 | 3-5% |

**布局示例（多肉花园第二关）：**
```
第1层（顶层，4个）：     [仙人掌] [芦荟] [景天] [熊童子]
第2层（中层，8个）：   [仙人掌] [芦荟] [景天] [生石花] [玉露] [仙人掌] [芦荟] [景天]
第3层（底层，12个）： [仙人掌] [芦荟] [景天] [生石花] [玉露] [熊童子] [仙人掌] [芦荟] [景天] [生石花] [玉露] [熊童子]
```

**难度控制策略：**

| 难度因素 | 设计 | 目的 |
|---------|------|------|
| 种类多 | 8种 vs 第一关4种 | 增加匹配难度 |
| 叠放层 | 3层 vs 第一关1层 | 遮挡制造不确定性 |
| 消除机会 | 每次滑动只移1格 | 需要精确规划 |
| 时间压力 | 无时间限制 | 让玩家自己给自己压力 |
| 失败惩罚 | 失败后重新开始 | 制造"再来一次"的冲动 |

**关键原则：**
- **第二关必须有解**——程序生成关卡后自动验证可解性
- 失败后显示"还差X株植物"，强化"就差一点"
- 连续失败5次后，道具使用免费1次（挽留用户）
- 连续失败10次后，提示"试试用放大镜看看？"（引导看广告）

### 2.4 关卡生成算法

**核心要求：每次生成的关卡必须有解**

```
算法流程：
1. 从当前主题的8种植物中随机选8种
2. 每种生成3个实例 → 共24个元素
3. 随机分配到3层（第1层4个、第2层8个、第3层12个）
4. 验证可解性：
   a. 从底层开始，找出所有可以直线相连的同种对
   b. 模拟消除，检查是否能全部消除完
   c. 如果无解 → 重新生成 → 直到有解
5. 输出最终布局
```

**验证方法：** 用回溯算法模拟玩家操作，如果能找到一条消除路径清空所有元素 → 有解。

### 3.3 摇花盆互动（趣味性提升）

**来源：抓大鹅的"颠锅"——物理互动让点击升级成有趣体验**

#### 操作方式

| 操作 | 效果 | 限制 |
|------|------|------|
| 摇手机 | 花盆里的植物重新随机排列 | 每天3次 |
| 滑动屏幕（向左/右） | 植物整体平移，露出边缘被遮挡的 | 无限次 |
| 点击"浇水"按钮 | 水从上方流下，冲走表层泥土，露出下面一层的植物 | 每局限1次（看广告可额外） |

#### 摇花盆的"坑"

像抓大鹅的颠锅一样，摇花盆也有策略性：
- **越摇范围越小**：连续摇2次以上，植物会向中间聚集，可选范围变小
- **摇完要立即选**：摇出来的植物3秒后恢复原位
- **不要乱摇**：有时候已经能看到的植物，摇完反而被盖住了

**设计意图：** 让玩家觉得"我摇到了好东西"而不是"系统帮我解决了问题"。

### 3.4 道具系统（广告变现核心）

#### 4种道具完整设计

| 道具 | 图标 | 功能 | 获取方式 | 使用限制 | 广告时长 |
|------|------|------|---------|---------|---------|
| 放大镜 | 🔍 | 高亮场上可匹配的3对植物（绿色箭头指向） | 看30秒广告 | 每局限1次 | 30秒 |
| 除草剂 | 🧪 | 清除场上3个随机障碍植物（爆炸特效） | 看30秒广告 | 每局限1次 | 30秒 |
| 洗牌 | 🔄 | 重新打乱剩余植物排列（旋转特效） | 看30秒广告 | 每局限1次 | 30秒 |
| 复活 | 💚 | 失败后继续当前局（从失败前3步恢复） | 分享链接 或 看广告 | 每局限1次 | 30秒（广告方式） |

#### 道具使用流程

```
玩家点击道具 → 弹出选择：
├── 分享链接给好友（社交裂变）
└── 观看30秒广告（变现）

玩家选择后 → 道具生效 → 返回游戏
```

#### 广告策略（分层控制）

| 阶段 | 广告策略 | 用户体验 |
|------|---------|---------|
| 第1-3局 | **不插广告**，道具免费使用 | 让玩家自由体验，快速上瘾 |
| 第4-10局 | 道具引导看广告，但不强制 | "看广告获得道具？"弹窗 |
| 10局以上 | 正常广告频率 | 每次道具=30秒广告 |
| 复活 | 优先引导分享链接，广告为备选 | "分享给好友获得复活" + "或看广告" |
| 连续失败5次 | 道具免费1次 | 挽留用户，防止流失 |

**避坑指南（简游2023年被工信部点名的教训）：**
- ❌ 不要强制看广告才能玩
- ❌ 不要关闭按钮藏在角落
- ❌ 不要广告自动播放
- ✅ 广告前明确告知时长
- ✅ 提供分享链接作为免费替代
- ✅ 前3局完全免费

#### 地域排名机制

- 每个玩家通关后，为自己的城市增加"绿化值"
- 绿化值 = 消除植物数 × 稀有度系数（普通×1，稀有×2，传说×5）
- 每天0点更新排名
- 玩家的城市通过IP或手动选择确定

### 6.1 IAA广告变现

## 九、避坑清单

| 坑 | 来源 | 我们怎么避 |
|----|------|-----------|
| 关卡无解被骂 | 羊了个羊 | 程序生成关卡后自动验证可解性（回溯算法） |
| 广告太激进被工信部点名 | 简游科技2023年 | 前3局不插广告，控制频率，提供分享替代 |
| 股权纠纷 | 简游（雷霆/吉比特） | 独立开发，不引入复杂股权 |
| 爆款后热度快速衰退 | 所有小游戏 | 持续更新关卡+新植物+每周主题 |
| 股权被冻结 | 简游2025年 | 合规经营，按时报税 |
| 纯消除疲劳 | 行业通病 | 每日主题+收集+互动三层驱动 |
| 抄袭争议 | 羊了个羊vs 3 Tiles | 滑动相邻消除≠折叠消除，玩法不同 |
| 包体过大 | 小程序限制 | 植物图片压缩到50KB以内，总包体<10MB |
| 加载慢 | 用户体验 | 预加载+懒加载，首屏<2秒 |
| 服务器宕机 | 羊了个羊 | 云开发弹性扩容，预留3倍容量 |

## 十、与AI植物管家的协同

| 协同点 | 说明 | 实现方式 |
|--------|------|---------|
| 内容复用 | 游戏中的植物照片/数据直接复用AI植物管家的数据库 | 调用同一API |
| 流量反哺 | 游戏是入口，AI植物管家是留存 | 卡片底部/通关页/花园页跳转 |
| 品牌统一 | "绿了个植"强化AI植物管家的植物专家定位 | 统一UI风格+品牌色 |
| 数据互通 | 游戏中高频出现的植物→AI植物管家推送相关养护内容 | 用户ID打通 |
| 冷启动 | AI植物管家的现有用户作为绿了个植的首批玩家 | App内入口+推送 |
| 内容反哺 | 游戏中玩家喜爱的植物→AI植物管家优先制作养护内容 | 数据分析驱动 |
